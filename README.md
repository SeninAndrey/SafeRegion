## ~~Задание №2~~

провести ревью и дать комментарии по компоненту [OptimizationList](./pages/review-list/review-list.tsx)

### обратить внимание

1) список динамический. как видно первый элемент зависит от значения value. 
доп вопрос: "если этот элемент будет точно первым или опследним, то  можно что-то сделать для оптимизации?"
2) список для примера всего из 9 элементов. в теории он бдует использоваться под 100 и более элементов

### Финальный результат

провести ревью компонента. основной упор сделать на оптимизацию и best-practice мобильной разработки
на react native. дополнительно можно дать комментарии и указать что не так, используя эту [рекомендацию](docs/recommendations.md)
как документацию к этому мини-"проекту". 

### Комментарии по улучшениям

1. **Динамический первый элемент title зависит от value**
    
    Проблема: При каждом изменении value создается НОВЫЙ МАССИВ data, что вызывает:
    - Перерисовку ВСЕХ 100+ элементов
    - Потерю состояния (key меняется для первого элемента)
    - Замедление ввода текста на слабых устройствах

    > **Решение: Мемоизировать базовый массив и динамически добавлять первый элемент**
***
2. **Ненадежный Key**

    Проблемы:
    - При дубликатах заголовков возникнет ошибка
    - При изменении value первый элемент меняет key → React пересоздает DOM-узел
    - Производительность падает при реконсиляции

    > **Решение: Использовать стабильный уникальный идентификатор**
***
3. **Отсутствие виртуализации для 100+ элементов(рендерятся все 100+ элементов сразу)**

    Проблема: 
    - 100+ элементов → 100+ View компонентов в памяти → просадка FPS, особенно на Android

    > **Решение: Использовать FlatList с виртуализацией**    
***

#### Ответ на дополнительный вопрос

> если этот элемент будет точно первым или последним, то можно что-то сделать для оптимизации?

да, для первого элемента:

```
// Использовать ListHeaderComponent в FlatList
<FlatList
    ListHeaderComponent={
        value ? <ListItem title={value} /> : null
    }
    data={BASE_DATA}
    renderItem={({item}) => <ListItem title={item.title} />}
/>
```

да, для последнего элемента:

```
// Использовать ListFooterComponent
<FlatList
    data={BASE_DATA}
    renderItem={({item}) => <ListItem title={item.title} />}
    ListFooterComponent={
        value ? <ListItem title={value} /> : null
    }
/>
```

<u>Преимущества такого подхода:</u>

1. Не нужно пересоздавать массив data при каждом изменении value

2. BASE_DATA остается стабильным → меньше перерендеров

3. FlatList может оптимальнее управлять виртуализацией

4. Экономия памяти: не хранится дублирующийся элемент в data массиве
